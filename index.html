Pyxel (/'pɪksəl/) est un moteur de jeu rétro pour Python. Avec des spécifications simples inspirées des consoles de jeux rétro, comme l'affichage limité à 16 couleurs et la prise en charge de 4 canaux audio, vous pouvez facilement créer des jeux au style pixel art. MediaPipe — Webcam & détection

MediaPipe détecte en temps réel le visage et les mains via la webcam. Les coordonnées sont accessibles dans votre code Python.

Prérequis : ajoutez import js en haut de votre code Python pour accéder aux variables MediaPipe.

Adapter les coordonnées à la taille de votre jeu

Les valeurs noseX, noseY et les coordonnées des mains sont normalisées entre 0.0 et 1.0. Pour les utiliser dans votre jeu, il faut les multiplier par la taille de l'écran Pyxel.

Par exemple, pour un écran de 160 x 120 pixels :

import js

LARGEUR = 160
HAUTEUR = 120

# Position du nez dans le jeu
x = js.window.noseX * LARGEUR
y = js.window.noseY * HAUTEUR

# Position de l'index de la main droite
ix = js.window.rightHand[8].x * LARGEUR
iy = js.window.rightHand[8].y * HAUTEUR

Adaptez LARGEUR et HAUTEUR aux valeurs passées à pyxel.init(largeur, hauteur).

Zone de détection (carré sur la webcam)

Sur la preview webcam, un carré en pointillés est affiché dans les 2/3 supérieurs de l'image. Il est défini par la ligne horizontale à 2/3 de la hauteur et deux lignes verticales centrées qui montent jusqu'en haut, formant un carré dont le côté vaut 2/3 de la hauteur de l'image.

Cette zone carrée est pratique pour mapper la position d'une main directement sur l'écran de jeu (qui est souvent carré).

Les coordonnées de cette zone en valeurs normalisées (0.0 à 1.0) sont :

  • Y : de 0.0 (haut) à 0.667 (2/3)
  • X : centré horizontalement, le côté du carré = 2/3 de la hauteur de l'image

Pour convertir la position d'un point (par exemple l'index de la main droite) dans cette zone en coordonnées de jeu :

import js

LARGEUR = 128
HAUTEUR = 128

# Limites de la zone de detection (valeurs normalisees)
ZONE_COTE = 2 / 3                     # taille du cote
ZONE_Y_MIN = 0.0                      # haut du carre
ZONE_Y_MAX = 2 / 3                    # bas du carre
ZONE_X_MIN = 0.5 - ZONE_COTE / 2     # bord gauche du carre
ZONE_X_MAX = 0.5 + ZONE_COTE / 2     # bord droit du carre

if js.window.rightHandDetected:
    # Position brute de l'index
    ix = js.window.rightHand[8].x
    iy = js.window.rightHand[8].y

    # Convertir en coordonnees de jeu (0 a LARGEUR/HAUTEUR)
    jeu_x = (ix - ZONE_X_MIN) / ZONE_COTE * LARGEUR
    jeu_y = (iy - ZONE_Y_MIN) / ZONE_COTE * HAUTEUR

    # Limiter aux bords de l'ecran
    jeu_x = max(0, min(LARGEUR, jeu_x))
    jeu_y = max(0, min(HAUTEUR, jeu_y))

Le point est ainsi mappé dans le carré de détection : quand la main est en haut à gauche du carré, les coordonnées valent (0, 0) ; en bas à droite, elles valent (LARGEUR, HAUTEUR).
